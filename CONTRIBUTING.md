# 贡献指南

感谢你考虑为家宴点菜小程序做出贡献！

## 如何贡献

### 报告 Bug

如果你发现了 bug，请在 GitHub Issues 中提交，包含以下信息：

1. **问题描述**：清楚地描述遇到的问题
2. **复现步骤**：详细的复现步骤
3. **期望行为**：你期望的正确行为
4. **实际行为**：实际发生的情况
5. **环境信息**：微信开发者工具版本、基础库版本等

### 提出新功能

如果你有新功能的想法：

1. 先在 Issues 中讨论你的想法
2. 等待维护者的反馈
3. 获得同意后再开始实现

### 提交 Pull Request

1. Fork 这个仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的改动 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

### 代码规范

- 使用 2 空格缩进
- 遵循微信小程序的开发规范
- 添加必要的注释
- 确保代码可读性

### 提交信息规范

使用清晰的提交信息，例如：

- `feat: 添加菜品评分功能`
- `fix: 修复购物车数量计算错误`
- `docs: 更新安装文档`
- `style: 优化菜单页面样式`
- `refactor: 重构订单查询逻辑`

## 开发流程

1. 克隆你 fork 的仓库
2. 配置云开发环境
3. 进行开发和测试
4. 提交 Pull Request

## 需要帮助？

如果你在贡献过程中遇到问题，请随时在 Issues 中提问。

再次感谢你的贡献！ 🎉
